📁 SCSS 디렉토리 구조 및 사용 가이드
우리 팀은 파견 구조로 운영되기 때문에 각 팀원이 혼자서도 프로젝트 구조를 이해하고 작업할 수 있도록
SCSS 파일을 모듈 단위로 나누어 구성했습니다. 아래는 디렉토리 구조 및 사용 원칙에 대한 설명입니다.
(Tailwind 스타일을 차용하여 공통 변수, 믹스인, 레이아웃 규칙 등을 정의합니다.)

──────────────────────────────────────────────────────────────────────────────────────────────────────────────────────────────────────────────────────

📂 1. 디렉토리 구조 설명

scss/
├── abstract/        👉 공통적으로 쓰이는 변수, 믹스인, 타이포 등 유틸
│   ├── _color.scss          // 컬러 변수 정의
│   ├── _index.scss          // abstract 하위 파일 통합 (import 진입점)
│   ├── _mixin.scss          // 믹스인 정의
│   ├── _root.scss           // CSS 변수나 기본 root 값 정의
│   ├── _typography.scss     // 글꼴 및 텍스트 관련 정의
│   └── _variables.scss      // 사이징, 간격 등 기타 변수들
│
├── base/              🧱 초기화, 전체 구조 정의
│   ├── _global.scss         // 전체 레이아웃, 공통 구조 설정
│   └── _reset.scss          // 브라우저 스타일 초기화
│
├── components/        🧩 헤더, 푸터 등 반복되는 UI 컴포넌트
│   ├── _footer.scss         // 페이지 하단 푸터 스타일 정의
│   └── _header.scss         // 상단 헤더 및 내비게이션 스타일 정의
│   
├── elements/        👉 버튼, 인풋 등 최소 단위 UI 요소
│
├── layout/          👉 페이지 레이아웃 관련 스타일
│
├── pages/           👉 페이지 단위 스타일
│
└── practice.scss    👉 실제로 사용하는 SCSS를 모아서 컴파일하는 메인 파일

💡 2. SCSS 사용 규칙
👉  모듈 시스템은 @use / @forward만 사용
    @import는 사용하지 않으며, SCSS는 모듈 기반 구조로 관리함.

👉  CSS 변수(:root) 우선 사용
    디자인 시스템(컬러, 폰트, 간격 등)은 :root에 바인딩된 CSS 변수를 기준으로 사용함.
    SCSS 변수($)는 내부 연산용 또는 계산 중간값에만 사용.

💡3. 변수 관리 방법

🎨 색상 시스템

1. 🎨 Base Color Palette – 브랜드 및 중립 색상 정의
_colors.scss에는 실제 색상값을 가진 기본 팔레트가 정의됩니다.
이 값들은 직접적으로 UI에 쓰지 않고, 시맨틱 토큰에서만 사용됩니다.

2. 🏷️ Semantic Tokens – CSS 변수로 사용되는 실제 UI 색상
@mixin color 안에 정의된 CSS 변수들은 실제 UI에서 사용하는 시맨틱 토큰입니다.
이 변수들은 base palette에서 가져온 색상을 역할 기반으로 매핑한 것입니다.

💡파일 작성 컨벤션

- 파일명: `_이름.scss` (언더스코어 필수)
- 변수명: `$gray-100`, `--color-text-weak` 형태
- mixin/함수: dash-case 또는 camelCase 통일
- 들여쓰기: 1 tab (4 space 크기)
